    STM32_Programmer_CLI 更多命令

    STM32_Programmer_CLI --help

    工具支持命令分类

    1. 连接目标板

    • -c, --connect 指定端口与设备建立连接（SWD、JTAG、UART、USB DFU、CAN、I2C等）

    -c port=SWD -c port=COM3

    2. 下载/烧录固件

    • -w, --write 烧录 bin/hex/elf 文件（可指定地址或直接烧hex/srec文件）

    -w app.bin 0x08000000 -w app.hex

    • -d, --download 功能同 -w，但命名为 download

    3. 擦除 Flash

    • -e, --erase [all|指定扇区]

    -e all # 擦除全部Flash -e [0 3] # 擦除扇区0到3

    4. 校验固件

    • -v, --verify 烧录后立即校验固件是否烧写成功

    5. 读取/导出 MCU Flash 内容

    • -u, --upload

    -u 0x08000000 0x10000 backup.bin # 从Flash读取内容保存为bin

    6. 复位/运行 MCU

    • -rst 软件复位
    • -hardRst 硬件复位
    • -g, --go [address] 跳转并运行

    7. 查看/操作 Option Bytes

    • -ob, --optionbytes [displ|unlockchip|OptByte=xxx]

    8. 读写内存

    • -r8/-r16/-r32 读内存
    • -w8/-w16/-w32 写内存

    9. 检查空白/校验

    • -blankcheck 检查Flash是否为空
    • -checksum, --checksum 校验某一段Flash内容

    10. 设备/调试相关操作

    • -list, --list 枚举所有连接的ST-Link/串口等设备
    • -l 31 输出详细调试信息（日志）

    其它高级/安全/MPU相关命令

    • 譬如 option bytes, TrustZone, Secure Boot, HSM, OTP, Provisioning, key、证书等管理，都在后面的高级命令和 Beta 命令里，普通应用/生产/量产基本用不上。

    自定义外部 Loader 的命令（后面用到再详细介绍制作过程）

    CubeProgrammer 使用自定义外部 Loader 的命令

    官方文档参数说明（关键部分）

    help 里的相关说明：

    -el,     --extload     : Select a custom external memory-loader for JTAG/SWD
     <file_path>        : External memory-loader file path

    作用： 让 CubeProgrammer 在烧录时，将自己制作的 loader（.stldr 文件）加载到 MCU 内存，再通过 loader 控制外部 Flash 烧录。

    命令用法举例

    假设你的外部 loader 文件叫 my_qspi_loader.stldr，你的固件叫 app_qspi.bin，你打算写到外部 QSPI Flash（典型起始地址如 0x90000000）：

    STM32_Programmer_CLI -c port=SWD -el my_qspi_loader.stldr -w app_qspi.bin 0x90000000 -v
    • -c port=SWD 连接的板卡
    • -el my_qspi_loader.stldr 指定自定义 Loader 文件（stldr 格式）
    • -w app_qspi.bin 0x90000000 写入 bin 文件到外部 Flash 的映射区（起始地址）
    • -v 烧录后校验

    Loader 文件（.stldr）制作方法见st官方仓库 https://github.com/STMicroelectronics/stm32-external-loader

    STM32_Programmer_CLI -c port=SWD -el path/to/your_external_loader.stldr -w your_flash_image.bin 0x90000000 -v
    • -el 或 --extload 后面跟的是你的外部 Loader 文件（.stldr），
    • -w ... 0x90000000 是你希望写到外部 Flash 的映射地址（比如 QSPI、OSPI 通常在 0x9000_0000 或 0x7000_0000）
